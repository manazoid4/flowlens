# Corporate buyer notes

Priority order (matches docs/product/buyer-personas.md): COO/Ops, Support/CX, QA/PM,
IT/MSP, Compliance/Audit, Training/Enablement, Agencies/Consultants, then Founders.

Buying triggers observed/assumed for this market:
- A failed audit or a near-miss compliance finding (Compliance/Audit trigger).
- A new hire ramp-up that took too long because documentation didn't reflect reality
  (Training, Ops trigger).
- A recurring class of support ticket that never got automated (Support, IT trigger).
- A bug that took multiple back-and-forth threads to reproduce (QA trigger).
- A client asking "how do you actually deliver this" for the first time (Agencies trigger).

Enterprise buyers will require (roadmap, not yet built): SSO/SCIM, data residency options,
custom retention policies, and a security questionnaire response — see docs/security
messaging and docs/deploy/supabase.md RLS note for what's real today vs. planned.
