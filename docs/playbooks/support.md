# Playbook: Support

**Goal**: reduce resolution time and standardize agent behavior on repeat ticket types.

1. Capture a resolution the first time a new ticket type is handled well.
2. Review the friction score — repeated copy/paste or manual lookups are your top automation
   candidates (see the refund-handling demo capture for a worked example).
3. Generate a training guide export and share it with the team as the canonical resolution
   path for that ticket type.
4. Route high-severity findings (e.g. a bug causing the tickets) to engineering directly
   instead of letting them recur silently.
